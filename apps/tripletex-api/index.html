NGD

Shelf detection

Official docs

Officially, NorgesGruppen Data was an object-detection task: detect and classify grocery products on shelf images, package the solver as a .zip, and run it in an offline NVIDIA L4 sandbox.

248 images ~22.7k boxes 356 categories 70 / 30 score split

Our strategy

Started with local Mac M4 Pro MPS runs for dataset prep, smoke tests, and ONNX packaging. Quality was too low to stay there, so the long training path moved to Google Colab L4 and G4 with YOLO11x, mirrored checkpoints, export sweeps, and runtime-safe threshold tuning before upload.

Result: 225 / 361, normalized 93.67.

Tripletex

AI accounting agent

Official docs

Officially, Tripletex asked teams to expose a public /solve endpoint that receives multilingual accounting tasks, optional attachments, and a fresh sandbox account, then completes the work through the Tripletex API.

30 task types 56 variants each 7 languages 5 min timeout

Our strategy

The core design was to keep the model bounded. LLMs extracted a constrained TaskSpec, but deterministic workflow modules owned API semantics, retries, ordering, and verification. Attachments were summarized first, then live fixes were driven by local scenario matrices and small verified batches.

Result: 188 / 412, normalized 46.59.

Astar

World-state prediction

Official docs

Officially, Astar Island was a stochastic simulation challenge: observe a 40×40 Norse world through a 15×15 viewport, spend a shared 50-query budget across 5 seeds, and submit a full probability tensor for the final terrain state.

40 × 40 map 5 seeds 50 shared queries KL divergence

Our strategy

The retained submission path stayed model-light and evidence-driven. Historical priors from completed rounds were updated with current viewport observations, the query budget was concentrated toward informative seeds, and guarded cross-seed refinement plus probability floors kept the KL score stable.

Result: 105 / 397, normalized 90.38.
